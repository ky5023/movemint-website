TransitEase は、新幹線が停車する全国主要ターミナルに特化し、多言語で人に寄り添う駅ナビゲーションを実現します。

ビジョン

わたしたちは「移動のストレスをなくす」ことを目指しています。 新幹線ダイヤに合わせて設計された在来線接続や駅動線の合理性を無駄にせず、複雑な構造・混雑・わかりにくい出口案内をオープンデータで見える化し、安心して移動できる世界をつくります。 訪日客が駅で足止めされる時間を地域体験へ転換し、日本各地での消費と回遊を後押しすることが狙いです。

課題

  • 駅構内の案内情報が統一されていない
  • Googleマップなどは「駅の中」を正確に案内できない
  • 高齢者や外国人、障がい者への配慮が不足
  • 実際の乗り換え時間とアプリ上の時間が合わない
  • 新幹線駅ごとの複雑さや文脈を反映した案内が提供されていない
  • 訪日客が長時間駅構内に滞留し、地域への回遊と消費機会が失われている

解決策（TransitEaseの仕組み）

01

駅構内ナビゲーション

オープンデータから改札・階段・エレベーター・出口の位置をマッピング。 新幹線駅の「中」まで迷わず案内します。

02

リアルタイム混雑回避

混雑情報API（例：東京メトロ）や人流推定データを活用し、混雑が少ないルートを優先提案。 オーバーツーリズムを防ぎ、駅周辺への回遊を促す移動をサポートします。

03

目的別ルート提案

「トイレに寄りたい」「エレベーターを使いたい」「人が少ない道を通りたい」など、 利用者の状況に応じた案内を提供します。新幹線発着時刻と接続ダイヤを加味し、滞在時間を周辺体験へブリッジします。

04

駅特化型AIコンシェルジュ

東京駅AIや京都駅AIなど、新幹線駅ごとにノーコードで構築可能なローカルAIを搭載し、 多言語でその時の気持ちや困りごとを会話ベースで解消してストレスを軽減します。 推奨スポットを分散提示することで、駅集中から地域全体への誘導も図ります。

オープンデータの活用

使用データ

  • GTFS / GTFS-RT（公共交通データ）
  • 駅施設データ（出口・改札・エレベーターなど）
  • 混雑情報API（JR東日本・東京メトロ・自治体オープンデータ）
  • OpenStreetMap / 国土地理院地図
  • 駅別ローカルAI用コーパス（東京駅AI・京都駅AIなど）

目的

  • 駅構内の経路情報を統合
  • 混雑を可視化し、分散を促す
  • 駅特化AIによるストレスの可視化と緩和
  • 観光や地域回遊の支援
  • オーバーツーリズム対策としての人流分散

価値提案

鉄道事業者向け

  • 混雑分散と利用者満足度の向上
  • リアルタイム人流データによる運行最適化
  • 駅特化AIによる問い合わせ削減と顧客対応の自動化
  • 新幹線接続ダイヤの有効活用による乗継率と滞在価値の向上
  • オーバーツーリズム抑制に向けた分散施策との連動

自治体向け

  • 観光案内や移動支援のデータ基盤に活用
  • 都市回遊データの取得と政策利用
  • 地域イベントに合わせたAIシナリオをノーコードで展開
  • 駅滞留時間を市街地消費へ誘導するオペレーション設計
  • 観光地の過密度を下げる多言語ナッジ設計

民間企業向け

  • 駅構内・出口付近の広告展開
  • ルート別ターゲティングの可能性
  • 駅別AIとの連携によるタイムリーな販促施策
  • 訪日客のリアルタイム意図を捉えたOMO施策
  • 混雑状況に応じた店舗アロケーションと動線最適化

チーム

TransitEaseは、交通データと人の行動理解を融合させるチームです。 UX設計・データ分析・都市交通の専門性に加え、駅特化型AIを迅速にノーコードで展開するケイパビリティを備え、 多言語対応と地域連携を組み合わせたオープンデータの新たな価値を提案します。
