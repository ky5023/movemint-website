交通オープンデータ × UX × AI

Navigate every Shinkansen station like a local

TransitEase は、新幹線が停車する全国主要ターミナルに特化し、多言語対応・3D構内案内・駅特化AIで移動体験を最適化します。

ビジョン

わたしたちは「移動のストレスをなくす」ことを目指しています。 新幹線ダイヤに合わせて設計された在来線接続や駅動線の合理性を無駄にせず、複雑な構造・混雑・わかりにくい出口案内をオープンデータで見える化し、安心して移動できる世界をつくります。 訪日客が駅で足止めされる時間を地域体験へ転換し、日本各地での消費と回遊を後押しすることが狙いです。

課題

  • 駅構内の案内情報が統一されていない
  • Googleマップなどは「駅の中」を正確に案内できない
  • 高齢者や外国人、障がい者への配慮が不足
  • 実際の乗り換え時間とアプリ上の時間が合わない
  • 新幹線駅ごとの複雑さや文脈を反映した案内が提供されていない
  • 訪日客が長時間駅構内に滞留し、地域への回遊と消費機会が失われている

解決策（TransitEaseの仕組み）

01

駅構内ナビゲーション

目的地検索で生成されたルートと3D構内モデルを同期し、改札・階段・エレベーター・出口までステップバイステップに案内します。 音声入力やクイックアクションで「途中でトイレに寄りたい」「別の出口にしたい」といった要望にも即応します。

02

リアルタイム混雑回避

混雑情報API（例：東京メトロ）や人流推定データを活用し、混雑が少ないルートを優先提案。 オーバーツーリズムを防ぎ、駅周辺への回遊を促す移動をサポートします。

03

ニーズ連動ルートリライト

検索時に最適ルートを提示したあとでも、チャットや音声で寄り道や体調に応じた変更を受け付けます。 新幹線発着時刻と接続ダイヤを加味し、滞在時間を周辺体験へブリッジします。

04

駅特化型AIコンシェルジュ

東京駅AIや京都駅AIなど、新幹線駅ごとにノーコードで構築可能なローカルAIを搭載し、 投稿された写真・テキスト・音声から現在地と文脈を推定して多言語で即時に困りごとを解決します。 推奨スポットを分散提示することで、駅集中から地域全体への誘導も図ります。

使い方イメージ

旅先で時間がない利用者を想定し、片手操作とスマートフォンの標準機能を最大限に活かした3ステップの体験を設計しています。

マイクボタンかクイック入力で目的地・到着時間をセット。途中で「Restroom break」など自然言語で伝えれば即時にルートへ反映。

オープンデータの活用

使用データ

  • GTFS / GTFS-RT（公共交通データ）
  • 駅施設データ（出口・改札・エレベーターなど）
  • 混雑情報API（JR東日本・東京メトロ・自治体オープンデータ）
  • OpenStreetMap / 国土地理院地図
  • 駅別ローカルAI用コーパス（東京駅AI・京都駅AIなど）

目的

  • 駅構内の経路情報を統合
  • 混雑を可視化し、分散を促す
  • 駅特化AIによるストレスの可視化と緩和
  • 観光や地域回遊の支援
  • オーバーツーリズム対策としての人流分散

価値提案

鉄道事業者向け

  • 構内3Dデータと動線シミュレーションによる signage / 改修意思決定
  • リアルタイム人流推定を用いた列車・ホームの混雑平準化
  • 駅特化AIによる多言語問い合わせの自動対応
  • 新幹線接続ダイヤと連動した乗継率・滞在価値の最大化
  • オーバーツーリズム抑制に向けた分散施策との連動

自治体向け

  • 観光案内・移動支援の多言語データ基盤として活用
  • 滞留ヒートマップと回遊ログによる政策立案
  • イベントや季節要因に応じたAIシナリオをノーコードで投入
  • 駅滞留時間を市街地消費へ誘導するオペレーション設計
  • 観光地の過密度を下げる動的なナッジ配信

民間企業向け

  • 駅構内・出口付近での高解像度ターゲティング広告
  • ルート別の動線と滞在時間に基づく店舗配置最適化
  • 駅別AIと連携したタイムリーなクーポン／プロモーション
  • 訪日客のリアルタイム意図を捉えたOMO施策
  • 混雑状況に応じたスタッフ配置・在庫出しの調整

チーム

TransitEaseは、交通データと人の行動理解を融合させるチームです。 UX設計・データ分析・都市交通の専門性に加え、駅特化型AIを迅速にノーコードで展開するケイパビリティを備え、 多言語対応と地域連携を組み合わせたオープンデータの新たな価値を提案します。
